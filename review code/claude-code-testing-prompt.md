# Claude Code 代码测试 Prompt 模板

## 一、通用代码审查 Prompt

```
请对我新写的代码进行全面审查和测试，具体步骤如下：

1. **代码理解**：先阅读项目结构和相关文件，理解这段新代码的功能意图。

2. **静态分析**：
   - 检查语法错误和类型问题
   - 检查变量命名和代码风格一致性
   - 检查是否有未使用的变量或引入

3. **逻辑审查**：
   - 检查边界条件是否处理（空值、零值、极大值、负数等）
   - 检查循环是否有死循环风险
   - 检查条件判断是否覆盖所有分支
   - 检查并发/异步代码是否有竞态条件

4. **安全检查**：
   - 检查是否有 SQL 注入、XSS 等安全隐患
   - 检查敏感信息是否暴露（密钥、密码等）
   - 检查用户输入是否做了校验和过滤

5. **性能检查**：
   - 检查是否有不必要的循环嵌套（时间复杂度）
   - 检查内存使用是否合理
   - 检查数据库查询是否有 N+1 问题

6. **生成测试**：为这段代码生成单元测试，覆盖以下场景：
   - 正常输入 (happy path)
   - 边界值
   - 异常输入和错误处理
   - 空值/null 处理

请先列出发现的所有问题（按严重程度排序：致命 > 严重 > 一般 > 建议），
然后给出修复方案和测试代码。
```

---

## 二、针对特定文件的快速检查 Prompt

```
请检查 [文件路径] 中我最近修改的代码：

1. 运行现有测试，看是否有测试被我的改动破坏
2. 分析我的改动可能引入的 bug
3. 为新增的函数/方法补充单元测试
4. 如果有测试失败，分析原因并给出修复建议
```

---

## 三、调试排错 Prompt

```
我的代码在运行时遇到以下错误：

[在此粘贴错误信息]

请帮我：
1. 定位错误发生的具体位置和根本原因
2. 分析导致这个错误的代码路径
3. 给出具体的修复方案
4. 确认修复后不会引入新问题
5. 添加测试用例防止此类问题复发
```

---

## 四、上线前最终检查 Prompt

```
这段代码即将上线，请做最终检查：

1. 运行所有相关测试，确认全部通过
2. 检查是否有遗漏的错误处理
3. 检查日志是否充分（关键操作是否有日志记录）
4. 检查配置项是否正确（环境变量、数据库连接等）
5. 评估这次改动的影响范围，是否会影响其他模块
6. 生成本次改动的简要总结（用于 code review）
```

---

## 使用建议

- **在 Claude Code 终端中**，你可以直接输入上面的 prompt，Claude 会自动读取你的项目文件
- 把 `[文件路径]` 替换成你实际的文件路径
- 可以根据你的技术栈（前端/后端/Python/Java 等）调整检查重点
- 建议在 git commit 之前跑一次「通用代码审查」，上线前跑一次「最终检查」

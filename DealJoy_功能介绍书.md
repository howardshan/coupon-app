> **DealJoy** — 北美 Dallas 地区本地生活团购平台
> 核心差异化：**"随时买，随时退"** — 无条件即时退款

## 一、产品架构概览

### 1.1 技术栈

| 层级 | 技术 |
|------|------|
| 用户端 / 商家端 | Flutter 3.x + Dart 3.11 |
| 状态管理 | Riverpod 2.6.1（AsyncNotifier 模式） |
| 导航 | go_router 14.3.0 |
| 后端 | Supabase（PostgreSQL + Edge Functions + Auth + Storage） |
| 支付 | Stripe（PaymentIntent + Webhook + Connect） |
| 扫码 | mobile_scanner 6.0.3 |

### 1.2 系统架构图

```
┌──────────────────┐    ┌──────────────────┐
│   用户端 App      │    │   商家端 App      │
│  (deal_joy)      │    │ (dealjoy_merchant)│
└────────┬─────────┘    └────────┬─────────┘
         │ 直接查表/RPC           │ Edge Function
         ▼                       ▼
┌─────────────────────────────────────────────┐
│              Supabase 后端                   │
│  ┌──────────┐  ┌────────────┐  ┌─────────┐ │
│  │PostgreSQL│  │16 个 Edge  │  │ Storage │ │
│  │  + RLS   │  │ Functions  │  │ (图片)  │ │
│  └──────────┘  └────────────┘  └─────────┘ │
│                      │                       │
│                ┌─────┴──────┐               │
│                │   Stripe   │               │
│                │  支付网关   │               │
│                └────────────┘               │
└─────────────────────────────────────────────┘
```

### 1.3 角色权限体系

```
品牌层（可选）
├── brand_owner   — 品牌创建者，一切权限
├── brand_admin   — 总部运营，管理所有旗下门店
│
门店层（核心）
├── store_owner   — 门店注册人/老板
├── manager       — 店长，管理该店全部功能（除设置）
├── service       — 客服，核销 + 订单 + 回复评价
└── cashier       — 核销员，只能扫码 + 看订单
```

## 二、用户端（Customer App）

### 2.1 认证系统

| 功能 | 说明 |
|------|------|
| 邮箱注册 | 用户名、全名、密码（≥8位，含大小写+数字）、服务条款 |
| 邮箱登录 | 邮箱 + 密码，支持"记住我" |

### 2.2 首页与浏览

**底部导航 4 个 Tab**：
1. **Deals** — 团购浏览（主页）
2. **Chat** — 消息（占位，V2）
3. **Cart** — 购物车（占位，当前为直接购买模式）
4. **Profile** — 个人中心

**首页功能**：
- **城市选择**：三级定位（State → Metro → City），如 Texas → DFW → Dallas
- **分类网格**：前 6 个热门分类图标
- **搜索模式切换**：在"搜门店"和"搜团购"之间切换
- **团购列表**：按选中城市浏览团购，下拉加载更多

### 2.3 搜索系统

| 功能 | 说明 |
|------|------|
| 热门搜索标签 | BBQ、Sushi、Hot Pot、Massage、Coffee 等 10 个 Dallas 本地热词 |
| 实时搜索建议 | 输入时防抖搜索，实时展示匹配结果 |
| 搜索历史 | 记录和展示历史搜索词 |
| 筛选排序 | 按距离、评分、折扣力度排序 |
| 分页加载 | 滚动到底自动加载更多 |

### 2.4 团购详情

| 评分评价 | 平均评分 + 评价数量 |

### 2.5 商家详情

商家详情页采用 **NestedScrollView + 5 个 Tab** 的设计：

#### Tab 1: Deals（团购）
- 顶部展示 Voucher 类型团购（特殊卡片样式）
- 分类筛选栏（商家自定义分类）
- 常规团购列表

#### Tab 2: Menu（菜单）
- **招牌菜**（Signature）：横向滑动大卡片
- **人气菜品**（Popular Picks）：小卡片网格
- **全部菜品**（All Items）：分类列表

#### Tab 3: About（关于）
- 环境照片（横向滑动）
- 便利设施标签（停车、WiFi、包间、大桌、预约、儿童椅、禁烟等）
- 商家简介描述
- 详细营业时间（周一至周日）

#### Tab 4: Reviews（评价）
- 评分统计：平均分 + 五星分布柱状图
- 评价列表：支持分页加载
- 每条评价显示：星级、评论内容、用户名、时间、照片
- 商家回复（如有）

#### Tab 5: Recommended（推荐）
- **同品牌其他门店**（连锁店）：显示品牌下的其他门店，点击跳转
- **附近推荐商家**：基于地理位置推荐附近商家

**其他功能**：
- 头图区域：照片轮播或三联图布局
- 品牌标识（连锁店显示品牌 Logo + "Chain" 标签）
- 收藏/分享按钮
- 底部操作栏：收藏、电话、导航、查看团购
- 照片画廊：2 列网格，按分类筛选（菜品、环境、其他、商家上传）

## 三、商家端（Merchant App）

### 3.1 认证与注册

#### 登录
- 邮箱 + 密码登录
- 角色校验（必须为 merchant 角色）
- 根据状态自动路由：
  - 无商家记录 → 注册流程
  - 待审核/被拒 → 审核状态页
  - 已通过 → 进入 Dashboard

#### 注册（5 步向导）
1. **账号创建**：邮箱、密码、确认密码
2. **商家信息**：公司名称、联系人、邮箱、电话
3. **分类选择**：Restaurant / Spa & Massage / Hair & Beauty / Fitness / Fun & Games / Nail & Lash / Wellness / Other
4. **证件上传**：营业执照、税号 (EIN)、身份证明
5. **地址提交**：街道地址、城市、州、邮编 + 同意条款

#### 审核状态页
- 显示"审核中"（预计 24-48 小时）
- 被拒绝时显示拒绝原因 + "修改并重新提交"按钮
- 手动刷新按钮检查状态

### 3.2 Dashboard（仪表盘）

**底部导航 4 个 Tab**：
1. **Dashboard** — 数据概览
2. **Scan** — 扫码核销
3. **Orders** — 订单管理
4. **Me** — 设置

**Dashboard 功能**：

| 模块 | 说明 |
|------|------|
| 问候语 | 根据时间显示 Good morning/afternoon/evening + 店名 |
| 上线/下线开关 | 一键切换门店在线状态 |
| 门店切换 | 品牌管理员可在多门店之间切换 |
| 今日统计 (2×2) | 今日订单数、已核销数、营收额、待处理数 |
| 快捷操作 (3×2) | 核销、团购、订单、评价、分析、门店、菜单 |
| 待办事项 | 待回复评价数、待处理退款数、Influencer 申请数（有数据才显示） |
| 7 天趋势表 | 日期 / 订单数 / 营收额，今日行高亮 |
| 下拉刷新 | 加载骨架屏 |

### 3.4 团购管理

#### 团购列表
| 功能 | 说明 |
|------|------|
| 4 个 Tab | 全部 / 进行中 / 已下架 / 审核中 |
| 分类管理 | AppBar 按钮打开分类管理面板（增删改分类） |
| 创建团购 | 浮动按钮进入创建页 |

#### 创建团购（5 步引导）
1. **基本信息**：标题、描述、使用须知、菜品选择、分类
2. **定价**：原价（菜品自动计算）、折扣价
3. **库存与有效期**：库存数量（或无限制）、有效期类型（固定日期/购买后 N 天）
4. **使用规则**：单人限购、可用日期（周一至周日）、可否叠加
5. **图片**：上传 1-5 张图片、拖拽排序、设置主图

## 五、核心业务流程

### 5.1 用户购买流程

```
浏览团购 → 点击"立即购买" → 选择数量 → 输入优惠码(可选)
→ 选择支付方式 → 创建 PaymentIntent → Stripe 支付
→ Webhook 回调 → 创建订单 + 生成券码(QR)
→ 支付成功页 → 查看券码
```

### 5.2 核销流程

```
用户出示券码(QR) → 商家扫码 → 验证券码有效性
→ 确认核销 → 更新状态为 used → 写入核销日志
→ 核销成功页面
（10 分钟内可撤销误操作）
```

### 5.3 退款流程

**用户主动退款**（随时退）：
```
未使用订单 → 选择退款原因 → 提交申请
→ Edge Function 调用 Stripe 退款 → 更新订单/券码/支付状态
→ 退款到原支付方式
```

**自动过期退款**：
```
券码过期 24 小时后 → Cron Job 触发 → 批量查询过期券码
→ 逐笔向 Stripe 发起退款 → 更新本地状态
（单笔失败不中断批次）
```

### 5.4 商家收款流程

```
用户支付 100% → 平台收取 15% 手续费
→ 商家实收 85% → T+7 天通过 Stripe Connect 打款
```
